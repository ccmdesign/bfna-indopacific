# feat: Add Embed Testing Page

## Overview

Create a dedicated testing page at `/test/embeds` that renders all available embed codes as live iframes, allowing developers and stakeholders to preview and verify every embeddable infographic in one place. The page should be excluded from production builds and search indexing.

## Problem Statement / Motivation

The project currently has two embed endpoints (`/embed/renewables` and `/embed/straits`), each wrapped in an iframe snippet generated by `useEmbedCode.ts`. Today, verifying these embeds requires:

1. Copying the embed code from the homepage or infographic page
2. Pasting it into a separate HTML file or external tool
3. Manually checking each embed one at a time

A single internal testing page that renders all embeds side by side would:

- Speed up QA after changes to embed pages or the embed layout
- Give stakeholders a quick visual check of all embeddable content
- Serve as a living reference for the exact iframe markup being generated

## Proposed Solution

Add a new Nuxt page at `pages/test/embeds.vue` that:

1. Imports the same infographic registry used on the homepage (`index.vue`)
2. For each entry, renders:
   - The infographic title
   - A live `<iframe>` pointing to `/embed/<slug>` (using the same dimensions and attributes as `useEmbedCode.ts`)
   - The raw embed code in a copyable `<code>` block
3. Uses the `default` layout with `robots: noindex, nofollow`
4. Is excluded from the Nuxt prerender list (not added to `nitro.prerender.routes`) so it does not ship in the static build

### Architecture

The page is intentionally simple: a single Vue SFC with no new components or composables. It reuses the existing `useEmbedCode` composable to generate the iframe snippets, ensuring the displayed code always matches what users actually copy.

```
pages/
  test/
    embeds.vue          <-- NEW: testing page
```

No new layouts, composables, or components are required.

## Technical Considerations

### Iframe Security Headers

The Netlify config (`netlify.toml`) sets `X-Frame-Options: DENY` and `frame-ancestors 'none'` on all routes except `/embed/*`. During local development (`nuxt dev`), these headers are not enforced, so iframes will load fine. On a deployed preview/staging build, the `/embed/*` routes already allow framing via `frame-ancestors *`, so the test page's iframes will work as long as the `src` uses the `/embed/` prefix.

**Key point:** The test page itself does NOT need to be frameable -- it only *contains* iframes that point to `/embed/*` routes.

### Static Generation (SSG)

The project uses `nitro.preset: 'static'` with explicit prerender routes. The test page should **not** be added to the prerender list. It is meant for local dev and deploy-preview usage only. If it is needed on a preview deploy, it can be added to prerender conditionally or accessed via SPA fallback (the `/* -> /index.html` redirect in `netlify.toml` already handles this).

### Reusing the Infographic Registry

The homepage (`pages/index.vue`) defines the infographic list inline:

```typescript
// pages/index.vue lines 26-39
const infographics: InfographicEntry[] = [
  { slug: 'renewables', title: 'Renewables on the Rise', ... },
  { slug: 'straits', title: 'Indo-Pacific Straits', ... }
]
```

The test page can either:
- **Option A (recommended):** Duplicate the slug list inline -- it is only two items and the test page is a dev tool, not user-facing. Keeps things simple.
- **Option B:** Extract the registry into a shared file (e.g., `data/infographics.ts`). This is cleaner but adds a refactor step that is out of scope for a test page.

The implementer should use Option A unless they prefer to refactor.

### useEmbedCode Composable

The `useEmbedCode` composable (`composables/useEmbedCode.ts`) generates the full `<iframe>` tag with proper dimensions (`width="1280" height="800"`, `aspect-ratio:16/10`) and HTML-escaped title attributes. The test page should use this composable to generate the embed code snippets, ensuring they always match production output.

## Acceptance Criteria

- [ ] A page exists at `/test/embeds` accessible during local development (`npm run dev`)
- [ ] The page renders a live iframe for each embed endpoint (`/embed/renewables`, `/embed/straits`)
- [ ] Each iframe uses the same dimensions and attributes as the `useEmbedCode` composable output (1280x800, border:0, aspect-ratio:16/10, loading:lazy, allowfullscreen)
- [ ] The raw embed code HTML is displayed below each iframe in a `<code>` or `<pre>` block
- [ ] Each embed code block has a copy button (reuse `EmbedCodeButton` or a simple copy-to-clipboard)
- [ ] The page has `robots: noindex, nofollow` meta tag
- [ ] The page is NOT added to `nitro.prerender.routes` in `nuxt.config.ts`
- [ ] The page uses the `default` layout and includes a "Back to home" link

## MVP

### pages/test/embeds.vue

```vue
<script setup lang="ts">
definePageMeta({
  layoutClass: 'layout-test',
  suppressRotateOverlay: true
})

useHead({
  title: 'Embed Testing - BFNA Indo-Pacific',
  meta: [
    { name: 'robots', content: 'noindex, nofollow' }
  ]
})

const embeds = [
  { slug: 'renewables', title: 'Renewables on the Rise' },
  { slug: 'straits', title: 'Indo-Pacific Straits' }
]

// Generate embed codes using the same composable as production
const embedCodes = embeds.map(e => {
  const { embedCode } = useEmbedCode(() => e.slug, () => e.title)
  return { ...e, code: embedCode }
})
</script>

<template>
  <main class="embed-test-page">
    <header>
      <h1>Embed Code Testing</h1>
      <p>Preview all embeddable infographics and their iframe codes.</p>
    </header>

    <section v-for="embed in embedCodes" :key="embed.slug" class="embed-test-section">
      <h2>{{ embed.title }}</h2>

      <div class="iframe-preview">
        <iframe
          :src="`/embed/${embed.slug}`"
          width="1280"
          height="800"
          style="border:0;max-width:100%;aspect-ratio:16/10"
          loading="lazy"
          allowfullscreen
          :title="embed.title"
        />
      </div>

      <div class="embed-code-block">
        <h3>Embed Code</h3>
        <pre><code>{{ embed.code }}</code></pre>
        <EmbedCodeButton :slug="embed.slug" :title="embed.title" />
      </div>
    </section>
  </main>
</template>
```

## Dependencies & Risks

- **No new dependencies.** The page only uses existing composables and components.
- **Risk: future embeds not added.** When a new infographic is created, the test page's inline list must be updated manually. This is acceptable for a dev tool. A comment in the file should remind the implementer.
- **Risk: SSG exclusion.** If someone adds `/test/embeds` to the prerender routes, the page will be deployed to production. The `noindex` meta tag mitigates SEO impact, but it should stay out of prerender.
